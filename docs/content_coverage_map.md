# 两本书内容覆盖地图

这份文档回答一个核心问题：

**两本书的哪些内容已经进入产品，哪些只抽了核心，哪些还没有正式课程化。**

## 当前覆盖结论

当前项目不是“整本书已经全部产品化”，而是：

1. 两本书资料都已入库
2. 两本书核心主线已经拆成 6 个模块
3. 已有 25 个核心概念、15 张重点图、6 个模块课程脚本
4. 重点覆盖已经集中在：
   - 《圆运动》的整体骨架
   - 《火神门》的扶阳主线
   - 两本书共享的方义、脉法、病例节奏

还没有完全覆盖的是：

1. 《圆运动》的中后段细化方解
2. 《圆运动》的生命宇宙篇世界观层
3. 《火神门》的完整医案细节与活解细则
4. 两本书逐章逐页的完整课程化

## 覆盖状态说明

- `已形成模块主线`：已经进入模块、概念、图表和课程脚本
- `已抽核心待扩写`：已经抽成概念或图表，但课程深度还不够
- `待进入产品化`：目前还只是资料层或摘要层

## 三、跨书整合情况

这两个地方已经形成了比较稳定的跨书整合：

1. `foundations`
   主要来自《圆运动》，但学习方法上已经吸收《火神门》的“先抓主轴”。

2. `formula-logic`
   用《圆运动》的方义和六经做骨架，同时吸收《火神门》的药性方向和扶阳招法语言。

3. `diagnostic-judgment`
   用《圆运动》的脉法、舌象做基础，再吸收《火神门》的“脉优先于病名”。

4. `fu-yang-core`
   主要来自《火神门》，但底层解释语言仍然依赖《圆运动》的整体观与升降出入。

5. `cases-and-application`
   两本书共享最多的地方：先后手、节奏、误治、活解。

## 四、当前空缺最大的区域

按优先级排序，当前最需要继续补的内容空缺是：

1. 《圆运动》伤寒论中下篇与金匮方解的杂病落地
2. 《火神门》医案辨析的完整病例拆解
3. 厥阴和复杂逆冲证的可视化
4. 温病与儿科模块的更具体案例练习
5. 生命宇宙篇的轻量化“世界观说明页”

## 五、下一阶段内容补全顺序

建议按这个顺序继续补：

1. 先把 `formula-logic` 的六经和方义拆得更细
2. 再把 `cases-and-application` 变成真正的病例推演区
3. 再补 `warm-disease-pediatrics` 的案例练习
4. 再决定要不要给 `生命宇宙篇` 单独做补充内容

## 六、这份覆盖地图怎么用

后面每次新增模块、图表、练习或病例时，都先回答 3 个问题：

1. 它来自哪本书、哪一部分？
2. 它现在属于哪个模块？
3. 它是在补已有主线，还是在新开一条支线？

只要这三个问题答不清，就不要急着做新页面。
